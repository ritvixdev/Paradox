▸Classification, regression

Generative AI
▸Generative models

▸Creates new content
▸Transformer (GPT, BERT)
ARCH Transformer Types
Encoder-only BERT — bidirectional, classification, NER, embeddings
Decoder-only GPT — autoregressive generation, next-token prediction
Encoder-Decoder T5, BART — seq2seq, translation, summarization
Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V
Complexity O(n²) time & memory — bottleneck for long sequences
Flash Attention Same math, GPU tiling trick — 2-4x faster, fits 10x longer seq
PARAMS LLM Sampling Controls
Temperature 0 = deterministic | 1 = creative. Scales logit distribution.
Top-K Only sample from K most probable next tokens (K=50 typical)
Top-P (nucleus) Sample from tokens whose cumulative prob ≥ P. Adaptive K.
Max Tokens Hard limit on output length. Controls cost.
Factual queries: temp=0, Top-P=1. Creative: temp=0.7–1.0

⑤ Fine-Tuning vs RAG Decision Framework
DECISION When to Use What
Factor Use RAG Use Fine-Tuning
Knowledge type External, factual, updatable Behavioral, stylistic, format
Data freshness Changes frequently Stable domain knowledge
Data availability No labeled training data Have domain examples
Output format Variable, natural language Strict schema / specific style
Hallucination risk Lower (grounded in docs) Higher (knowledge in weights)
Update cost Re-index documents (cheap) Re-train model (expensive)
Inference latency +retrieval latency overhead Lower (no retrieval step)
Best of both: Fine-tune for domain terminology + output format. Add RAG for grounding in facts. This is the production pattern for enterprise AI.
METHODS Fine-Tuning Techniques
Full Fine-Tune All weights updated. Risk: catastrophic forgetting.
LoRA Train small rank-decomposed matrices A·B alongside frozen weights. 10,000x fewer params.
QLoRA LoRA + 4-bit quantization. Fine-tune 65B model on 1 consumer GPU.
DPO Skip reward model — directly train on (preferred, rejected) pairs.
LoRA: W' = W₀ + α·(A·B) — only A, B trained

MATH Distance Metrics
Cosine Sim Angle between vectors. Range -1 to 1. Best for text — magnitude-invariant.
Dot Product Similar to cosine but magnitude matters. Use with normalized vectors.
Euclidean (L2) Geometric distance. Less common for NLP embeddings.
cos(θ) = (A·B) / (|A|·|B|)

⑦ Evaluation & Metrics
EVALS Gen AI Evaluation Pyramid
Infrastructure Latency (TTFT), throughput, error rate, cost/request
RAG Quality Faithfulness + Answer Relevance + Context Relevance (RAGAS)
LLM-as-Judge GPT-4/Claude scores outputs on rubric — practical at scale
Human Eval Domain experts — non-negotiable for medical/legal/regulated domains
METRICS Precision vs Recall Trade-off
Precision = TP / (TP + FP) — of flagged, how many correct?
Recall = TP / (TP + FN) — of all real, how many caught?
F1 = 2·P·R / (P+R) — harmonic mean
High Recall Catch all cheaters (accept some false accusations)
High Precision Only flag when certain (miss some cheaters)
Use cost ratio C(FP)/C(FN) to determine operating threshold

⑨ ML Fundamentals (Data Science Round)
ML Bias vs Variance
High Bias = Underfitting
▸Model too simple
▸Poor train + test accuracy
▸Fix: more features, complex model
High Variance = Overfitting
▸Model memorizes noise
▸Good train, bad test
▸Fix: regularization, more data
ML Random Forest
▸Ensemble of decision trees — each trained on a random bootstrap sample of data (bagging)
▸Each split considers only a random subset of features (feature randomness)
▸Prediction = majority vote (classification) or mean (regression)
▸Reduces variance without increasing bias — uncorrelated errors cancel
ML R² vs Adjusted R²
R² = 1 - SS_res / SS_tot [0 to 1, higher = better]
Adj R² = 1 - (1-R²)(n-1)/(n-p-1) [penalizes extra features]
▸R² always increases when you add features — even if they're noise
▸Adjusted R² decreases if new feature doesn't improve the model
▸Always use Adjusted R² when comparing models with different feature counts

⑫ 60-Second Interview Answers
ANSWER What is RAG? (30 sec)
"RAG grounds LLM responses in external documents. Offline: chunk docs → embed → store in vector DB. Online: embed query → cosine similarity search → inject top-K chunks into LLM prompt. This solves hallucination and knowledge cutoff without retraining."
ANSWER What is Agentic AI? (30 sec)
"An agent uses an LLM as a reasoning engine that autonomously plans, calls tools, and iterates until a goal is achieved — not just responding to single prompts. Key patterns: ReAct (think-act-observe loop), tool use via function calling, and multi-agent orchestration with planner + executor separation."
ANSWER Fine-Tune vs RAG? (30 sec)
"RAG for knowledge, fine-tuning for behavior. RAG when you need access to updatable external facts — easy to maintain. Fine-tune when you need consistent output format, domain terminology, or task-specific style that doesn't change frequently. Best production pattern: fine-tune for behavior + RAG for factual grounding."
